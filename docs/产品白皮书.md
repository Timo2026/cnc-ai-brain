## 一、产品概述

**Union·由你** 是一款运行在工控机上的离线 AI 决策系统，专为 CNC 加工企业解决"接不接单、报什么价、工艺能不能做"三大核心决策问题。

传统模式：销售拿图纸 → 问工程师能不能做 → 翻手册报价 → 找老板拍板 → 2小时到2天。

**Union·由你**：输入需求 → 系统自动召开 CFO + BI + 工艺总监 + CEO 多专家会议 → 10秒内输出报价 + 工艺可行性 + 接单决策 + 审计记录。

**一句话**：把 CNC 工厂的"接单决策链"从人工接力变成 AI 自动化。

## 二、核心能力

### 2.1 智能报价
- **引擎**: 基于 20,000+ 条制造知识库的 rule-based 引擎，非 LLM 凭空猜测
- **精度**: 材料成本 × 加工费率 × 数量梯度 × 表面处理系数 × 利润
- **速度**: <1 秒/单
- **覆盖**: 304/316L/6061/7075/钛合金/45钢/碳钢等 50+ 材料，阳极氧化/镀锌/电镀/喷砂/抛光等

### 2.2 多专家决策会议
| 专家 | 职责 | 数据来源 |
|------|------|----------|
| CFO | 财务分析：利润率、回款周期、现金流 | 精确报价引擎 |
| BI 分析师 | 客户历史：复购率、信用等级、过往订单均价 | SQLite 订单库 |
| 工艺总监 | 工艺可行性：公差、材料禁忌、加工难度 | 10 条硬规则 + LLM |
| 战略官 | 战略价值：新客户开拓、产能匹配 | LLM 语义分析 |
| CEO | 最终裁决：综合决策、一票否决/覆写 | 以上全部汇总 |

**决策机制**：
- CFO 拥有一票否决权（利润率 <5% 或现金流风险）
- CEO 可覆写否决（留痕审计）
- 所有决策过程写入 SQLite 哈希链，不可篡改

### 2.3 工艺冲突检测
- **硬规则**: 10 条离线规则（如：304+阳极氧化=不可能、6061+淬火=不可行）
- **软规则**: LLM 语义分析兜底（检测未覆盖的新材料组合）
- **拦截**: 工艺不可行 → 直接阻断报价，给出修正建议

### 2.4 离线审计
- 每次报价、每次专家会议生成 SHA-256 哈希记录
- 链式审计：每条记录含上一条哈希，篡改必留痕迹
- 支持 `/api/audit` 公开验证

## 三、技术架构

```
┌─────────────────────────────────────────────┐
│              Web UI / REST API              │
│         FastAPI + HTML (localhost:7861)      │
├─────────────────────────────────────────────┤
│  NeuroCore — 串行多专家引擎                  │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌─────┤
│  │ CFO  │→│  BI  │→│工艺  │→│战略  │→│ CEO │
│  │💰报价│ │📊历史│ │🔍检查│ │🎯战略│ │⚖️裁决│
│  └──────┘ └──────┘ └──────┘ └──────┘ └─────┤
├─────────────────────────────────────────────┤
│  AI Engine — Ollama (qwen2.5:3b, 全本地)     │
├─────────────────────────────────────────────┤
│  Skill Layer — 真实工具调用                   │
│  quote.py · conflict_check · history_lookup  │
├─────────────────────────────────────────────┤
│  Safety — SQLite 哈希审计链                   │
├─────────────────────────────────────────────┤
│  AutoAdapt — 环境检测 · 模型选择 · 技能注册    │
└─────────────────────────────────────────────┘
```

**关键设计原则**：
- **零硬编码**: 所有路径/参数/模型由运行时自动推导
- **全离线**: 无外网依赖，数据不出车间
- **自适应**: 根据 CPU/GPU/内存自动调整专家数量和模型选择
- **不可篡改**: 哈希链审计，满足工业合规

## 四、部署方式

### 方式 A：源码运行（最小依赖）
```bash
pip install -r requirements.txt
python app/main.py
# 浏览器打开 http://localhost:7861
```

### 方式 B：Docker 一键部署
```bash
docker compose up -d
# 自动拉取 Ollama + qwen2.5:3b + 启动服务
```

### 方式 C：Windows 离线部署
```bash
# 在开发机上构建 EXE（需 Windows + Python）
pyinstaller union_by_ni.spec
# 复制到目标工控机，双击 start_windows.bat
```
**前提**: 目标机安装 Ollama + `ollama pull qwen2.5:3b`

## 七、客户场景

### 场景 1：销售接单
> 销售拿客户图纸，微信问老板"能不能做"。老板不在，等了 3 小时。客户去找别人了。

**Union·由你**: 销售输入"6061法兰 100件 阳极氧化"，系统 1 秒返回报价 ¥12,300，无工艺冲突。

### 场景 2：高价值订单决策
> 钛合金叶轮 20 件，客户报价 15 万。接不接？

**Union·由你**: CFO 算成本 ¥10.3 万（利润 31%）→ BI 查客户历史（复购 3 次，付款准时）→ 工艺确认 IT6 可做 → CEO 批准。80 秒完成决策。

### 场景 3：工艺冲突拦截
> 客户要求 304 不锈钢+阳极氧化。新手销售差点接单。

**Union·由你**: 直接阻断。"304 不可阳极氧化，建议钝化"。避免了一次 5 万元的质量事故。
